Scroll Animations

Scroll down — each card animates into view, driven purely by CSS.

🎨

Animation Timeline

CSS animation-timeline: view() links an animation's progress directly to an element's position in the scroll viewport.

⚡

No JavaScript

No IntersectionObserver, no event listeners. The browser handles everything declaratively based on scroll position.

🎯

Animation Range

animation-range lets you define exactly when in the scroll journey the animation starts and ends.

🧩

Composable

Combine with CSS custom properties and @keyframes to create complex staggered reveal effects without any library.

🚀

Performant

Scroll-driven animations run on the compositor thread, meaning zero layout recalculations and butter-smooth 60fps scrolling.
